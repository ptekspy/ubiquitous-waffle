# v0.4 main-branch improvement pass

This pass turns the repo from a working prototype into a safer product foundation.

## Backend and codebase changes

1. Product-ops tables are now represented by Prisma migrations instead of runtime request-handler DDL.
2. Browser import and product-ops mutation routes validate inputs before doing parsing or writes.
3. Browser import requests have a configurable payload limit via `IMPORT_PAYLOAD_LIMIT_BYTES`.
4. Planner worker processing recovers stale `RUNNING` jobs using `PLANNER_STALE_LOCK_MS`.
5. Structured `EventLog` rows capture scan imports, rejected imports, planner events, and planner worker failures.
6. CI now uses `pnpm install --frozen-lockfile`, generates Prisma client, typechecks, lints, runs Node tests, checks the extension contract, and builds.
7. Node static-contract tests protect the import route, product-ops route, migration ownership, planner recovery, and extension privacy contract.
8. The extension now has a typed protocol contract in `extension/protocol.ts` so the JS service worker can be migrated gradually without changing the runtime bundle yet.

## Product/app changes

9. Workspace health now includes a scan-quality score derived from raw/cleaned rows, warnings, and truncation metadata.
10. A `GET /api/product/next-actions` endpoint ranks the next useful actions from scan state, best subreddit, deep-dive backlog, planner failures, planned posts, tracked subreddits, and peers.
11. The dashboard now renders a **Next actions** card so the app tells the user what to do next instead of only showing analytics.
12. Product ops summary now surfaces scan quality as a first-class health item.
13. The extension static check ensures Manifest V3 is preserved and the extension does not request cookie access.
14. Event logging is ready for a future activity feed/debug panel.
15. The migration adds queue-supporting indexes for planner locks and deep-dive job ownership/status.

## New environment options

```bash
IMPORT_PAYLOAD_LIMIT_BYTES="5242880"
PLANNER_STALE_LOCK_MS="900000"
```

## Pull/deploy note

After pulling, run:

```bash
pnpm install --frozen-lockfile
pnpm db:generate
pnpm db:migrate
pnpm verify
```

For production, use:

```bash
pnpm db:deploy
pnpm verify
```
